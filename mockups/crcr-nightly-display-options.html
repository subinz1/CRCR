Option A: Page-Level Tabs

Three tabs at the top — PR, Nightly, Periodic. Each shows the same layout filtered by event type.

+ Pros

  • Clean, one view at a time
  • Familiar GitHub tab pattern
  • Each tab can have its own stat cards

- Cons

  • Can't see all types at once
  • Extra click to switch

Option B: Health Matrix

Single table — repos as rows, event types as columns with colored dots + percentages.

+ Pros

  • Everything visible at once
  • Quick at-a-glance health check
  • Very compact

- Cons

  • Dense with many repos
  • Many "—" cells if repos don't run all types

Option C: Expandable Repo Cards

Each repo is a card with colored event-type pills visible at a glance. Click to expand for per-type details.

+ Pros

  • Overview pills visible without clicking
  • Details on demand
  • Natural grouping by repo

- Cons

  • Requires interaction for details
  • Expanded cards take more space

Option D: Side-by-Side Columns

Three columns — PR, Nightly, Periodic — each listing only repos that participate. Like a Kanban board for CI health.

+ Pros

  • Clear visual separation
  • No N/A entries
  • Easy to see participation

- Cons

  • Same repo duplicated across columns
  • Tight on narrow screens

Option E: Swimlane Timeline

  • Run frequency visible
  • Matches main HUD grid pattern

- Cons

  • Needs more horizontal space
  • More complex to implement
