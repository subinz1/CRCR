CRCR Summary — Nightly/Periodic Display Options

6 different approaches for displaying nightly and periodic CI alongside PR results

Option A: Page-Level Tabs

Recommended

Three tabs at the top of the page — users click to switch between PR, Nightly, and Periodic views. Each tab shows the same table layout but filtered by event type. Familiar pattern from GitHub, Jira, etc.

Pros

  • Clean, uncluttered — one view at a time
  • Familiar UI pattern (GitHub tabs)
  • Each tab can have its own stat cards
  • Easy to deep-link (?tab=nightly)

Cons

  • Can't see all event types at once
  • Extra click to switch views

Option B: Health Matrix

A single unified table where each row is a repo and columns show health status for PR, Nightly, and Periodic. Everything visible at a glance — colored dots indicate health.

Pros

  • Everything visible at once — no tab switching
  • Quick at-a-glance health check
  • Compact and information-dense

Cons

  • Can feel dense with many repos
  • Less room for per-event-type details
  • Many "N/A" cells if repos don't run all types

Option C: Expandable Repo Cards

Each repo is a card with event-type pills visible at a glance. Click to expand and see detailed stats for each event type (PR, Nightly, Periodic) in a 3-column grid.

Pros

  • Overview pills visible without clicking
  • Details on demand — doesn't overwhelm
  • Natural grouping by repo

Cons

  • Requires interaction to see details
  • Expanded state takes more vertical space

Option D: Side-by-Side Columns

Three columns side by side — Pull Requests, Nightly, Periodic — each listing repos that participate in that event type. Like a Kanban board for CI health.

Pros

  • Clear visual separation of event types
  • Easy to see which repos participate in which
  • No wasted space on N/A entries

Cons

  • Same repo appears in multiple columns
  • Horizontal space constrained on narrow screens
  • Harder to compare a single repo across types

Option E: Swimlane Timeline

A timeline view showing recent runs as colored cells. Each repo has sub-rows for PR, Nightly, and Periodic. Great for spotting trends, intermittent failures, and run frequency at a glance.

Pros

  • Shows trends over time (flaky vs. persistent failures)
  • Run frequency immediately visible
  • Matches main HUD grid mental model

Cons

  • Takes more horizontal space
  • More complex to implement
  • May feel unfamiliar to some users

Option F: Stacked Sections with Visual Bars

Three vertically stacked sections on the same page — each with a distinct colored header. All visible without switching, with progress bars for pass rates. Each section only shows repos that participate in that event type.

Pros

  • All event types visible by scrolling
  • No N/A entries — only relevant repos shown
  • Progress bars add visual clarity
  • Clear visual distinction between sections

Cons

  • Longer page — requires scrolling
  • Same repo duplicated across sections
